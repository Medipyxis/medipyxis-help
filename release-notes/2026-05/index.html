May 2026 release notes

release-note Clinician Intake Admin Billing Business development Vendor Updated 2026-07-08

Changes shipped between 2026-05-01 and 2026-05-31. Items are grouped by module. Entries marked ★ are high-impact or frequently requested.

Visit Wizard / EHR

  • ★ Visit Wizard — Soft warnings replace hard blocks. Steps that previously hard-stopped on incomplete fields now surface a soft warning and let the clinician proceed ("warn and proceed"). Silent failures are now visible warnings. (PR #2333, #2318)
  • ★ Visit Wizard — "Sign & Lock" close-out action. The close-out call-to-action was renamed from Save & Complete Review to Sign & Lock. (PR #2325)
  • ★ Visit Wizard — Smart AI Import. A new Smart Import button extracts wound and visit details from an uploaded document or photo and pre-fills wizard fields, with a banner listing anything skipped. (PR #2302, #2303, #2308)
  • ★ Visit Wizard — Pull from History. Bring forward documented sections (ROS, comorbidities, functional status, exam, care plan, education, etc.) from prior visits, with preview-and-select. (PR #2360, #2350)
  • ★ Visit Wizard — Pull Migration Data. A Pull Migration Data button was added to the top-of-wizard toolbar in both wizards, alongside Reset Form. (PR #2311, #2309)
  • ★ Wound status — Treating vs. Monitoring. Each wound carries a Treating / Monitoring pill; Monitoring wounds skip rules that apply to actively treated wounds. (PR #2314)
  • Visit Wizard — "Running HPI" label. The free-text Session Context Notes field was renamed Running HPI. (PR #2300)
  • Visit Wizard — Resume Visit shortcut in the wound cockpit. A Resume Visit card is anchored first in the cockpit when entering from the Fleet Calendar. (PR #2271, #2272)
  • Visit Wizard — Open-visits docket. An inline docket of other open visits lets clinicians switch between in-progress visits. (PR #2365, #2270)
  • Visit Wizard — Admins can see in-progress drafts. (PR #2267)
  • Visit Wizard — Wizard timer and Backup/Restore UI removed. Legacy timer and manual Backup/Restore retired; false "unsaved changes" warning fixed. (PR #2307, #2305)
  • Visit Wizard — Build version badge. Always-visible build-version badge (bottom-left) + stale-build refresh prompt. (PR #2352, #2263)
  • Visit Wizard — Topical Application Site shows body location. (PR #1973)
  • Visit Wizard — Save toast no longer sticks. (PR #1971)
  • Visit Wizard — Clinical history carry-forward. (PR #1869)

Clinical Workflow & Hand-off

  • ★ Clinical Dashboard rebuilt with hand-off workflow. Status tiles, a work queue, and a stale-visit view. Providers can Hand off a visit to support staff and Mark Visit Completed. New Awaiting Documentation status. (PR #2012, #2009)

Referral Intake

  • ★ Consent — SMS now sent on demand. The patient consent SMS is no longer auto-sent; triggered by a manual Send button. (PR #2331)
  • Intake — Unified manual and fax referral creation. Both now flow through the same shared records service. (PR #1936)
  • Intake — Insurance drawer shows unverified coverage. (PR #1937, #1940)

Billing & Claims

  • ★ Billing — ClaimMD fully replaced by Stedi. Claims and eligibility now run through Stedi, including enrollment and eligibility verification. (PR #1895, #1889, #2010)
  • ★ Eligibility — Stedi verification with clearer errors. Errors now show the human-readable description. (PR #2000, #2013)
  • ★ Billing — Wound-care code catalog expanded. CPT debridement/destruction ranges and additional HCPCS Q-codes, seeded from CMS LCDs. (PR #2346, #2293, #2294, #2268)
  • Billing — Manual code entry persists and is searchable. Multiple codes per wound; searchable picker. (PR #2266, #2269)
  • Billing — ICD-10 auto-fill in Generate Codes. (PR #1964)
  • Billing — Quantity/units on CMS-1500. (PR #2298)
  • Billing — NPI/MBI validation at submit. (PR #1870)

Patient Management & Scheduling

  • Patient Management — Default status filter aligned with Fleet Calendar (All). (PR #2295)
  • Patient Management — "Needs Scheduling" cleanup. Single place, auto-default provider, merges twin appointments. (PR #2322)
  • Scheduling — Same-day scheduling supported; provider-filter fix. (PR #1883)

Inventory

  • ★ Inventory — Consolidated product catalog (catalog_v2). PRP/UM modeled as inventory categories. (PR #1929, #1927)
  • Inventory — Vendor dropdowns point to the CRM vendor hub. (PR #2320)

Fax

  • Fax — Cover page settings and audit trail. (PR #1882)
  • Fax — Settings access for support and admin roles; Send Fax visibility fix. (PR #1984, #2005)

DrChrono Import

  • DrChrono — Documents and wound cases surface in the chart. (PR #1857, #1856)
  • DrChrono — Faster document import. (PR #1843)

HR & Admin

  • HR — Edit User Profile unblocked (via admin-update-user); user-creation surfaces the real error on failure. (PR #1985, #1955, #1951)

Mobile (iOS)

  • Mobile — Daily Schedule buttons clear iOS Safari chrome. (PR #1942)

Documents & Progress Notes

  • Progress Notes — Accuracy fixes. No hardcoded CHF from a parent flag; debridement-method fallback fixed; blank-screen fixed. (PR #1972, #1970, #1877)

Infrastructure / Security

  • Security — Tightened data access policies across patient, billing, inventory, org data; administrator area locked to Medipyxis-internal users. Back-end safeguards; no workflow change. (PR #1958, #1962, #1963, #1960, #1934)
  • Build — Stale-build detection with one-click refresh. (PR #2285, #2291)

User-facing changes from the next deploy cycle are documented in June 2026 release notes.

June 2026 release notes

release-note Clinician Intake Admin Billing Business development Vendor Updated 2026-07-08

Changes shipped between 2026-06-01 and 2026-06-25. Items are grouped by module. Entries marked ★ are high-impact or frequently requested.

ZusHealth & TEFCA Integration

  • ★ ZusHealth — Patient history sync rebuilt. Click-driven importer covering allergies, medications, lab results, surgeries, vaccinations, and conditions. The Pull button in pre-charting drives the new sync. (PR #2370)
  • ★ ZusHealth — Sync status and source badges. Last-sync pill (green/yellow/red) and per-row provenance badges. (PR #2370)
  • ★ ZusHealth — Automatic visit contribution (TEFCA). On visit finalize, Medipyxis contributes the visit back to the patient's external record via ZusHealth/TEFCA — no extra step. (PR #2706)
  • ZusHealth — Comprehensive Patient History added to pre-charting, plus reliability fixes. (PR #2687, #2697, #2698)

Visit Wizard / EHR

  • ★ Visit Wizard — Offline mode removed from the web app. The legacy wizardSafetyNet offline system was removed. (PR #2711)

If your team relied on in-wizard offline/backup behavior, coordinate with your implementation manager on the current save model. The wizard now saves continuously with per-field auto-save.

  • ★ Visit Wizard — DME product picker removed from the DME section. (PR #2707)
  • ★ Visit Wizard — Cross-device draft recall. Drafts are recalled by a stable visit identifier, eliminating wrong-wound data and cross-device load failures. (PR #2700, #2701, #2704)
  • ★ Visit Wizard — Reliable per-field auto-save. The 60-second interval was replaced with per-field debounced saving plus a heartbeat. (PR #2553)
  • Visit Wizard — Secondary payer read-only in the wizard; edits on the Insurance tab; written back to the canonical insurance record. (PR #2705)
  • Visit Wizard — Open-visits docket moved to the top; older Pull from History / View Previous Visits removed in favor of the new pull workflows. (PR #2687, #2682)
  • Visit Wizard — eRx labeled DoseSpot; calls routed through a server-side proxy. (PR #2662)
  • Visit Wizard — Imported wound history in the cockpit (two-pane); migration-context banner for transferred patients. (PR #2642)
  • Visit Wizard — Native camera on mobile web for wound photos. (PR #2637, #2638)
  • Visit Wizard — Fewer false error toasts and several data-loss/silent-failure fixes. (PR #2631, #2674)

Clinician Dashboard

  • ★ Dashboard — Overdue undocumented visit alerts in a dedicated card with disposition actions. (PR #2695)
  • Dashboard — Real data sources (placeholder data removed). (PR #2650)

Billing & Claims

  • ★ Charge Master access for billers. The biller role can view and edit the facility Charge Master; billing roles are facility-scoped (no global wildcard). (PR #2710)
  • ★ Billing Work Queue — Filters, bulk actions, metrics. Status/payer/age filters, bulk status changes, status dropdown with history, Coding Review drawer, dashboard metric cards. (PR #2559, #2561, #2558)
  • ★ Billing — Auto-populate codes for claim-ready visits; de-dup by date of service. (PR #2639)
  • ★ Billing — Patient superbill PDF download + preview on the claim detail page. (PR #2601)
  • ★ Eligibility — Full Stedi medical-catalog sync with typeahead payer picker, resubmit cooldown, and Medicare MBI gating. (PR #2532, #2606)
  • Billing — Drag-to-reorder claim lines with CMS-1500 Box 24 re-assertion. (NIM-150)
  • Billing — Facility NPI/Tax ID validation and NPPES auto-fill. (PR #2607, #2614)
  • Billing — Place of Service defaults to Home (POS 12). (PR #2557)
  • Billing — Only documented supplies billed (auto-fill of saline A4216 removed). (PR #2625)

Referral Intake

  • Intake — Carry-forward across 13 sections; progress-note preview; unified finalize + claim auto-creation (parity with Visit Wizard). (PR #2627, #2628)
  • Intake — Referral field renames: Referral Date → Referral Entry Date; Referral Source Date → Date Referral Sent. (PR #2587)
  • Intake — Cross-device draft recall. (PR #2704)

Fleet Calendar

  • ★ Fleet Calendar — Clinician identity unified across Fleet Calendar, HR, and chat; top scroll nav added; display bugs fixed. (PR #2629, #2678)
  • Fleet Calendar — Off-hours visits visible. (PR #2556)
  • Scheduling — Non-clinician appointment creation via provider dropdown, stricter provider/NPI validation. (PR #2627, #2554)

Vendor / CRM

  • Vendors — Lists repointed to the CRM vendor hub; dead practice.vendor_* lookups and stale category sentinels removed. (PR #2707, #2708, #2709)

Inventory

  • ★ Inventory — Free-form bulk upload (no purchase order required), promoted to the page header. (PR #2591, #2550)
  • ★ Inventory — FDA HCT/P recall lookup view and page. (PR #2580)
  • Inventory — New conservative product types (Surgical Instrument, PPE, General Supply). (PR #2535)
  • Inventory — Forensic audit trail (immutability triggers, FDA chain-of-custody). (PR #2579)

Practice Onboarding (Admin)

  • ★ Practice Onboarding — New admin hub with guided patient import + chart-data import modules (wounds, visit history). (PR #2642)

Team Chat & Notifications

  • ★ Team Chat — Unified chat with case linking and Kudos threads; consolidated notification pipeline. (PR #2649, #2652, #2653, #2654)

Business Development

  • Business Development — Dashboard scoped to the logged-in rep; load race condition fixed. (PR #2651)

Documents & Progress Notes

  • ★ Progress Notes — Zero-fabrication safeguards. Clinical-entity and medication scrubbers with zero-fabrication constraints; fabricated plan items removed from the PDF. (PR #2622, #2623)

Infrastructure

  • Analytics — Product usage instrumentation (PostHog) and error tagging (Sentry), behind the scenes. (PR #2541, #2545)
  • Reliability — Saves are unconditional; DB safeguards against duplicate visits and double-billing. (PR #2611, #2612, #2613, #2643)
